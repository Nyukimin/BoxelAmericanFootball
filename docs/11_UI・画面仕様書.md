# 11_UI・画面仕様書

- 目的: 教材性とゲーム性を両立させるための画面、HUD、操作、情報表示を定義する
- 関連: `01_教材コンセプト仕様書.md`, `03_学習カリキュラム仕様書.md`

## 1. 基本方針

UIは、本作がマネージャー教材として成立するかを左右する。

長い説明文を出すのではなく、試合を見るために必要な情報を、試合画面の中で自然に読めるようにする。

UIの目的:

- いま何が起きているか分かる
- 次に何を選べばよいか考えられる
- プレー結果を記録として読める
- 用語を初出で理解できる
- 詳しく知りたいときだけ深掘りできる

## 2. 画面構成

基本画面には次を配置する。

| 領域 | 内容 |
|---|---|
| メインビュー | フィールド、選手、ボール、アニメーション |
| HUD | ダウン、残りヤード、スコア、シリーズ、攻撃権 |
| カードUI | 隊形、プレー、守備隊形、守備戦術 |
| コーチメッセージ | 結果や用語の一言解説 |
| マネージャーメモ | 記録、見るポイント、次の判断 |
| ミニマップ | ボール位置、フィールド全体 |
| 用語ポップアップ | 初出用語の短い説明 |
| リプレイ操作 | もう一度見る、スキップ、次へ |

## 3. HUD

### 3-1. 必須表示

| 表示 | 例 | 目的 |
|---|---|---|
| ダウン | `2nd` | 何回目の攻撃か分かる |
| 残りヤード | `& 6` | ファーストダウンまでの距離 |
| 攻撃権 | `味方攻撃` | どちらの番か分かる |
| フィールド位置 | `敵陣35yd` | 陣地と得点判断 |
| スコア | `味方 7 - 3 相手` | 勝敗状況 |
| シリーズ | `3/6` | 試合の残り感 |
| 目標ライン | 光るライン | 10ヤード地点を見せる |

### 3-2. 表示例

```text
味方攻撃 | 2nd & 6 | 敵陣35yd | シリーズ 3/6 | 味方 7 - 3 相手
```

### 3-3. 教材UIとしての注意

- `2nd & 6` だけでなく、初期は「2回目 / 残り6ヤード」も併記できる
- 慣れてきたら専門表記へ寄せる
- 目標ラインを光らせ、言葉を読まなくても状況が分かるようにする

## 4. カードUI

### 4-1. 隊形カード

カードには次を表示する。

- 隊形名
- 可愛いアイコン
- 見た目の並び
- 一言説明
- 向いているプレー

例:

```text
ショットガン
パス向き。QBが後ろに構えて投げやすい。
```

### 4-2. プレーカード

表示項目:

- プレー名
- リスク
- 見返り
- 状況に応じたヒント

例:

```text
ショートパス
成功しやすく、少し進みやすい。
```

### 4-3. 守備カード

守備フェーズでは、守備隊形と戦術を2段階で選ぶ。

例:

```text
ラン重視
中央を厚くして、走る攻撃を止めやすい。
```

```text
ブリッツ
強く前へ出る。奪えるかも。でも外すと危ない。
```

## 5. プレー後表示

プレー終了後は、教材として重要な情報を短く表示する。

### 5-1. 必須項目

- プレー記録
- 次の状況
- 結果理由
- 見るポイント
- 次の判断材料

### 5-2. 表示例

```text
記録: シングルバック / ラン +4yd
状況: 2nd & 6
見るポイント: ラインが少し押して、走る道ができた
次の判断: 残り6yd。ランもパスも選べる場面
```

### 5-3. 長さ制限

- 1行は短めにする
- 詳細説明はリプレイまたは用語図鑑へ逃がす
- 毎回長文にしない

## 6. コーチメッセージ

マスコットコーチは、結果とルールを一言で解説する。

### 6-1. 方針

- 必要なときだけ
- 短く
- できれば理由を含める
- 用語はやさしく言い換える

### 6-2. 例

| 場面 | メッセージ例 |
|---|---|
| ファーストダウン | 「10ヤード進めたから、新しい4回のチャンスだよ。」 |
| 3rd & short | 「あと少し。ここは確実に進みたい場面だね。」 |
| パント | 「得点は狙わず、相手を遠くから始めさせるキックだよ。」 |
| 反則 | 「黄色いフラッグ。今のは反則だよ。」 |
| テーピング | 「ピョン太をテーピング中。今回はランが少し不利かも。」 |

## 7. マネージャーメモ

記録・分析マネージャーの存在を活かし、試合記録の見方を示す。

### 7-1. 表示例

```text
マネージャーメモ:
ラン +4yd。中央が詰まったけど、少し押し込めた。
次は2nd & 6。
```

### 7-2. 役割

- 記録の型を見せる
- 見るべきポイントを教える
- 次の判断へつなげる
- リプレイへの導線にする

## 8. 用語ポップアップ

初出用語のみ、短いポップアップを出す。

### 8-1. 表示内容

- 用語名
- やさしい説明
- 用語図鑑へ登録されたこと

例:

```text
用語: ファーストダウン
10ヤード進めたので、また4回のチャンスがもらえること。
用語図鑑に追加しました。
```

### 8-2. 方針

- 自動で消える
- 任意で開き直せる
- プレー中の邪魔にならない

## 9. リプレイUI

プレー後に、任意でリプレイを見られる。

### 9-1. ボタン

- もう一度見る
- スキップ
- 次へ

### 9-2. 表示内容

- スロー再生
- 各選手の役割ラベル
- ボール軌跡
- ライン攻防
- 守備の動き
- 結果の理由

### 9-3. 方針

- 強制しない
- 学びたい人だけ見る
- いつでもスキップ可能
- あとから一覧で見返せるようにする

## 10. 視点UI

### 10-1. クオータービュー

基本視点。

- 全体を見渡す
- 並びとボール位置を見る
- 采配画面に使う

### 10-2. トップビュー

正確な位置関係を見る。

- 目標ライン
- ヤード
- 陣形
- ボール位置

### 10-3. 選手視点

迫力を味わう観戦モード。

- ボールキャリアの背後
- 任意切替
- 教材上は必須ではない

### 10-4. キックビュー

FGミニゲーム用。

- ボール後方からゴールを見る
- ボール横位置を反映する
- 正面固定にしない

## 11. ボールを見失わない工夫

- 常時光るマーカー
- ボール軌跡
- ミニマップ
- パス/キック時の放物線
- ファンブル時の目立つ跳ね

## 12. スマホ・PC対応

### 12-1. PC

- マウスクリック
- キーボードショートカットは任意

### 12-2. スマホ

- タップ操作
- カードは大きめ
- 文字量は少なめ
- FGミニゲームは2段階タップ

### 12-3. 共通

- 誤タップしにくい配置
- ボタン間隔を確保
- 画面端に重要情報を寄せすぎない

## 13. 試合終了画面

表示項目:

- 最終スコア
- 勝ち/負け/引き分け
- 主な記録
- 見つけた用語数
- 「もう一度あそぶ」

例:

```text
試合終了
味方 14 - 10 相手
見つけた用語: 12個
もう一度あそぶ
```

## 14. UIの優先実装

教材価値を上げる優先順:

1. HUDの分かりやすさ
2. プレー後の記録表示
3. コーチの一言
4. 用語初出ポップアップ
5. リプレイ導線
6. マネージャーメモ
7. 用語図鑑
8. 守備テル表示
9. FGミニゲームUI

## 15. 非対象

- 長文チュートリアル画面中心の設計
- 複雑なメニュー階層
- 画面を止める強制学習
- 上級者向け戦術ボードの初期実装
- 細かすぎる公式記録システム
