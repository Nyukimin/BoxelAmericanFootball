# 仕様追記：守備の采配（味方が守備のとき）

- 作成日: 2026年6月15日
- 状態: 実装済み（TDD→ヘッドレスE2E検証済み）。ブラウザ目視は未確認。
- 位置づけ: 企画書 第12章「守備の采配は後から追加」の実装
- 関連: gamelogic.js（defenseDrive/pickCpuStyle）／index.html（攻守交代フロー）

## 1. 目的
相手の攻撃を「確率だけ」から「味方の守備采配（隊形→戦術）＋守備stats＋読み合い」に格上げし、攻撃と守備の両輪にする。攻撃と同じ2段階カードで初心者でも選べる。

## 2. 体験（味方が守備のとき）
攻守交代で守備の番になり、2段階で選ぶ：
- 守備隊形: ラン重視 / バランス / パス重視 / ゴールライン
- 戦術: 通常 / ブリッツ / ゾーン
結果: 止める(0) / FG許す(相手+3) / TD許す(相手+7) / ターンオーバー奪取(味方ボール)。1相手ポゼッション＝守備采配1回。

## 3. 読み合い
CPUは隠れ傾向(run/pass/balanced)を持つ。隊形が刺さると大きく止める、外すと通される。
- runStop↔run、passDef↔pass、goalLine↔run(特に強・深いパスに弱い)、balanced=中庸。
- blitz: ターンオーバー奪取率↑（外すと手薄）。zone: 大失点(TD)を抑える堅実。
- 守備stats: power=ラン阻止/奪取、speed=カバー/インターセプト。

## 4. 純粋ロジック（gamelogic.js）
- pickCpuStyle() -> "run"|"pass"|"balanced"。
- defenseDrive({front,tactic,los,pinBonus,dPower,dSpeed,cpuStyle}) -> {points:0|3|7, turnoverWon, msg}。
  - base=陣地、counter=隊形と相手傾向の読み合い、statRed=守備能力、tactic（blitz=奪取/手薄、zone=TD抑制）で goodness を補正し、TD/FG/ストップを抽選。係数は初期値（調整可）。

## 5. 統合（index.html）
- state に mode("offense"|"defense") / cpuStyle / defLos / defPin を追加。
- 攻守交代 → endPossession が守備フェーズを起動 → chooseDefenseUI（守備隊形4枚）→ pickDefenseTactic（戦術3枚）→ resolveDefense（defenseDriveで判定）。
- turnoverWon なら味方ボールで良い陣地から攻撃再開、else state.away += points。drive 消化で規定到達なら試合終了、そうでなければ攻撃へ。
- 旧 opponentPossession は削除（守備采配に置換）。
