# 仕様追記：フィールドゴール（FG）ミニゲーム

- 作成日: 2026年6月14日
- 状態: 設計（未実装）。本ドキュメントは実装前の仕様。
- 位置づけ: 企画書「ボクセル・フットボール（仮称）」への追記
- 関連: `docs/仕様追記_タックル_ライン_ファンブル_キック演出.md`（キック演出）／第5章（視点）

## 1. 目的

確率任せだったフィールドゴールを、プレイヤーが**左右の角度と飛距離を自分で狙う**ミニゲームにする。ボールを置いた位置からの正しい見え方で、横からのキックほど難しくなる駆け引きを出す。

## 2. 操作（2段階タップ）

採用: 2段階タップ方式。

- ステップ1【左右の角度】: 角度マーカーが左右に往復。タップで止めて狙う向き（`aim`）を決める。中央に近いほどまっすぐ。
- ステップ2【パワー（飛距離）】: ゲージが伸びる。タップで止めて飛距離を決める。距離に届くパワーが必要。

```
左 [-----|----▲----|-----] 右   ← 1回目タップで角度
弱 [#######...] 強               ← 2回目タップでパワー
```

決定後、ボールが狙った向き・強さで放物線で飛ぶ。ポールの間を通れば成功＝3点。

## 3. ポールの見え方（最重要・正面固定にしない）

- FG中のカメラは**ボールの少し後方・低めから、ゴールポストを見る**。視線方向 = ボール位置 → ゴール中心。
- これにより、**ボールが横（ハッシュ寄り）にあると、ゴールは画面の左右どちらかに寄り、斜めに見える**。正面固定にはしない。
- 横にあるほど「通せる左右の角度の窓」が狭く・片寄って見え、難度が上がる（＝左右角度が意味を持つ）。

```
中央から:            左サイドから:
   |     |               |   |     ← ゴールが右に寄り斜め＝窓が狭い
   |     |             .....\
   |  O  |             O------\...→ 狙う向き
  まっすぐで広い         斜めに通す必要
```

実装の勘所: `updateCamera` に "kick" ビューを追加し、`camera.position = ballPos + (0, 約6, 約-14)`、`lookAt = ゴール中心`。ボールの x がそのまま見え方に反映される（特別な補正は不要）。

## 4. ゴールポストの配置（見た目と判定で共用）

- 位置: エンドゾーン奥。`z = 120` の平面に H 型ポール。
- 左右アップライト: `x = ±3`（通すべき幅）。クロスバー高さ `y ≈ 3`、その上に支柱が伸びる。
- `field()` にゴールポストのオブジェクトを追加し、上記座標を判定にも使う。

## 5. 成否判定（幾何ベース）

ボール位置 `(xb, zb)`（zb = 現在の `los`、xb = 直近プレーのボール横位置。無ければ 0。ハッシュ相当に `[-6, 6]` でクランプ）。

- 各ポールへの方位角:
  - `θL = atan2(-3 - xb, 120 - zb)`
  - `θR = atan2( 3 - xb, 120 - zb)`
- プレイヤーの角度 `θaim` が `θL <= θaim <= θR` なら左右は成功圏。
  - 横にあるほど `[θL, θR]` が片寄り・狭くなる（角度が効く）。
- パワー: 必要飛距離 = ボール→ゴール距離（`120 - zb` を基準に+α）。`power` がそれ以上なら飛距離OK。
- 成功 = 左右OK かつ 飛距離OK。失敗は理由で出し分け:
  - 左右外: 「右にそれた！／左にそれた！」
  - 飛距離不足: 「届かなかった…！」

ボールは `θaim` 方向・`power` の初速で**実際にその軌道で飛ばす**（見た目と判定を一致させる）。

## 6. 能力・天候の反映

- `kick` 能力（6が平均）: 角度メーターの往復速度を下げる／成功窓に少し許容（tolerance）を足す＝上手いほど易しい。
- 天候（任意・将来）: 風の強さ・向きを `θaim` にオフセットとして加える（横風でそれる）。ゲーム性仕様の天候候補と連動。

## 7. 演出

- キッカーがボールを蹴る短い動作 → 放物線で飛ぶ（既存パスの放物線を流用）。
- 成功: ポールの間を通過 → 3点、スコア演出。
- 失敗: 外側へそれる／手前で失速して落ちる。
- いずれも痛々しくしない（第7章）。

## 8. 調整パラメータ（初期案）

| 項目 | 初期値/式 | 備考 |
| --- | --- | --- |
| ゴール幅 | `x = ±3` | 通すべき左右幅 |
| ゴール奥行 | `z = 120` | 判定平面 |
| 角度メーター可動域 | `θcenter ± 0.28 rad` | 窓より広めに |
| メーター速度 | 基準 × `(2 - kick/6)` | kick高いほど遅く=易しい |
| 成功窓の許容 | `+0.01 × (kick/6)` | わずかに広げる |
| ハッシュ範囲 | `xb in [-6, 6]` | キック原点の横位置 |

## 9. フロー（既存への接続）

1. 4thダウンで「フィールドゴール」を選ぶ（既存）。
2. カメラを kick ビューへ。角度メーター表示 → タップ。
3. パワーメーター表示 → タップ。
4. ボールが飛ぶ → 幾何で成否確定 → メッセージ。
5. 成功なら `state.home += 3`、いずれも `endPossession()` で攻守交代（既存）。
