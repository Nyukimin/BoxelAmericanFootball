# 仕様追記：けが・テーピング補助（低頻度ランダム／采配連動）

- 作成日: 2026年6月14日
- 位置づけ: 企画書「ボクセル・フットボール（仮称）」への追記
- 関連章: 第7章（アート方針：痛々しくない可愛い動き）／第8章（学習設計：必要なときだけ伝える）／第3章（采配）／第4章（采配システム）

## 1. 目的・背景

マネージャーのサポート業務のひとつ「けが・テーピング補助」を、ゲーム画面上に低頻度の演出として可視化する。ぶつかって転ぶことのあるアメフトで、選手をケアして送り出すマネージャーの役割を、可愛く・痛々しくない形で伝える。あわせて、テーピング中の選手の担当プレーを一時的に不利にすることで、「選手のコンディションも采配の判断材料になる」ことを学べるようにする（第4章・第8章と連動）。

## 2. 仕様

- 発生タイミング: 攻撃が継続する穏やかな場面（ファーストダウン獲得・通常前進の後）にランダム判定する。ターンオーバー・タッチダウン・攻守交代などの慌ただしい場面では発生させない。
- 頻度制御（多発させない）:
  - 発生確率は低め（初期値15%）。
  - クールダウンを設け、一度発生したら最低数プレー（初期値3プレー）は再発生しない。
  - 既にテーピング中の選手には重ねて発生させない。
- 演出内容:
  - 攻撃側の動物選手1体に、白いテーピング（ボクセルの帯）が付く。
  - マスコットコーチが一言フォローし、不利になるプレーを伝える（例:「ピョン太 をテーピング中。今回はランがちょっと不利かも。ほかの手もあるよ。」）。
  - 痛々しい表現はしない（第7章準拠。転んでもポヨンと可愛く）。
- ゲーム性への影響（軽め・采配連動）:
  - テーピング中の選手の担当プレーが、その「次の1プレーのみ」少し不利になる。
    - 走る役（ランバック）／ライン … ランの獲得ヤードが減る。
    - 司令塔（QB）／受け手（レシーバー）… パスの成功率・獲得ヤードが下がる（ロングはインターセプト率も微増）。
  - 効果は1プレーで終わり、プレー後にテーピングが外れて通常状態に戻る（回復）。
  - あくまで軽い影響にとどめ、初心者がテンポよく遊べる範囲に抑える。選手が完全に抜ける・退場するといった強い処理は行わない。

## 3. 既存仕様との関係

- 第3章（コアループ＝采配）: 変更なし。テーピングは采配の対象ではない。
- 第7章（アート方針）: 「痛々しくない可愛い動き」に沿った見せ方。
- 第8章（学習設計）: 「必要なときだけ一言」の方針に沿い、発生時のみコーチが短く触れる。
- 記録・分析マネージャー（別追記）と合わせ、マネージャーのサポート業務を段階的にゲームへ織り込む位置づけ。

## 4. 実装状況

- `index.html` に実装済み。
  - `maybeTape()`: 攻撃継続時（`nextPlayReady`）に確率・クールダウン判定し、ランダムな攻撃選手に白いテーピング帯を付与。役割（QB／ランバック／レシーバー／ライン）を記録。
  - `outcome()`: テーピング中の役割に対応するプレー（ラン／パス）の結果を軽く不利に補正。
  - `recoverTape()`: `finalize()` の先頭で呼び、1プレーで帯を外して通常状態へ回復。
  - `chooseFormationUI()`: 発生時、隊形選択メッセージに不利プレーを伝えるコーチの一言を付加。

## 5. 調整パラメータ（実装上の初期値）

| パラメータ | 初期値 | 意味 |
| --- | --- | --- |
| 発生確率 | 15% | プレー継続時に判定する確率 |
| クールダウン | 3プレー | 再発生までに空けるプレー数 |
| 効果の持続 | 1プレー | テーピングが効いている（不利になる）プレー数 |
| ラン不利 | 獲得ヤード×0.55 | 走る役／ラインが手当て中のとき |
| パス不利 | 失敗率↑＋獲得ヤード×0.7 | QB／レシーバーが手当て中のとき |

いずれも「多くなりすぎない・効きすぎない」ことを優先して低めに設定。手応えを見て調整可能。

## 6. 今後の拡張余地（任意・後段）

- テーピングを軽いミニ操作（ボタンで手当て→復帰）にする。
- サポート業務5「コンディション／スタミナ管理」と接続し、疲労がたまるとテーピング頻度が上がる等の連動。

いずれも企画書の方針（初心者向けにルールを絞る／テンポ優先／学習を強制しない）に沿い、まずは演出として小さく入れてから検討する。

## 7. 制約・非対象（明示）

- ゲームへの影響は「次の1プレーだけ軽く不利」に限定する。選手の退場・長期離脱・強いペナルティは行わない（初心者向けのシンプルさを維持）。
- 守備側・相手チームのテーピングは対象外（攻撃側のみ）。
- 原本（企画書.docx）は変更せず、本ドキュメントを追記として扱う。
- 見た目（テーピングの位置・大きさ・タイミング）と効きの強さはWebGL実機での確認・調整が必要。
