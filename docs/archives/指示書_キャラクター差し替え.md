# 指示書：キャラクターの仕様と差し替え方法

- 作成日: 2026年6月14日
- 対象読者: キャラクター（選手・マネージャー）を追加・変更する担当者
- 関連: 企画書 第7章（アート方針・データ駆動）／`characters.js`

このゲームのキャラクターは「データ（設定）」と「見た目（ボクセルのパーツ）」を分けて管理しています。**ほとんどの差し替えは `characters.js` の設定リストを書き換えるだけ**で完了します。プログラム本体（`index.html`）は触りません。

---

## 1. ファイル構成と役割

| ファイル | 役割 | 編集するか |
| --- | --- | --- |
| `characters.js` | キャラ定義（設定リスト＋見た目ビルダー） | **ここを編集する** |
| `index.html` | ゲーム本体（ルール・采配・描画） | 原則さわらない |

`characters.js` は読み込まれると `window.BoxelChars` を公開し、`index.html` がそれを使ってキャラを並べます。

---

## 2. 選手は「3つの独立要素」でできている

1選手 = 役割（role）＋ 見た目（animal/fur）＋ パラメータ（stats）。
この3つは独立していて、自由に組み合わせられます（例：足が速い・くま・パンター）。

```js
{ name: "ダッシュ", role: "P", animal: "bear", fur: FUR.bear, stats: "speedy" }
```

| 項目 | 意味 | 例 |
| --- | --- | --- |
| `name` | 表示名（コーチのセリフ等で使われる） | `"ミミィ"` |
| `role` | 役割（ポジション）。下の一覧から選ぶ | `"QB"` |
| `animal` | 見た目の動物。下の一覧から選ぶ | `"owl"` |
| `fur` | 毛色。`0xRRGGBB` 形式の数値 | `0xe7d4a8` |
| `stats` | パラメータ。プリセット名 または直接オブジェクト | `"speedy"` |

### 使える `role`（役割）

| 値 | 役割 |
| --- | --- |
| `"QB"` | クォーターバック（司令塔・パスを投げる） |
| `"RB"` | ランニングバック（走る役） |
| `"WR"` | レシーバー（受け手） |
| `"OL"` | ラインマン（守る役・押し込む） |
| `"P"` | パンター（蹴る役） |
| `"DF"` | ディフェンス（守備チーム用） |

### 使える `stats`（パラメータ）プリセット

`speed`（足の速さ）/ `power`（パワー）/ `catch`（捕球）/ `kick`（キック）を 1〜10 で持つ。
プリセット名で指定すると使い回せる。直接 `{ speed: 7, power: 2, catch: 8, kick: 1 }` のように書いてもよい。

| プリセット名 | 中身（speed/power/catch/kick） | 特徴 |
| --- | --- | --- |
| `"speedy"` | 9 / 4 / 6 / 3 | 足が速い |
| `"power"` | 4 / 9 / 5 / 3 | パワー型 |
| `"hands"` | 6 / 4 / 9 / 3 | 捕球がうまい |
| `"kicker"` | 5 / 5 / 4 / 9 | キックが得意 |
| `"allround"` | 6 / 6 / 6 / 6 | バランス型 |

※現在 `stats` のうちゲームに効くのは `speed`（ランの走る速さの見た目）。他は今後の拡張用に保持。

### 使える `animal`（見た目）の種類

| 値 | 動物 | 見た目の特徴 |
| --- | --- | --- |
| `"owl"` | フクロウ | 大きな目の縁・くちばし・羽角 |
| `"rabbit"` | ウサギ | 長い直立耳（内側ピンク）・前歯 |
| `"squirrel"` | リス | 房毛耳・ほっぺ・大きな縞しっぽ |
| `"bear"` | クマ | 丸耳・茶色マズル・黒鼻 |
| `"rhino"` | サイ | 角・小さい耳・大きめマズル |
| `"wolf"` | オオカミ | とがった耳（先黒）・白マズル・眉・白先しっぽ |
| `"cat"` | ネコ | 三角の耳（内側ピンク）・ピンク鼻・ひげ・縞しっぽ |

上記以外の文字列を指定すると、汎用（丸耳＋鼻）の見た目になります。

## 3. チーム編成リスト（`characters.js` 内）

```js
// 攻撃チーム（味方）。各選手 = name / role / animal+fur / stats
var OFFENSE = [
  { name: "ミミィ",   role: "QB", animal: "owl",      fur: FUR.owl,      stats: "allround" },
  { name: "ピョン太", role: "RB", animal: "rabbit",   fur: FUR.rabbit,   stats: "speedy" },
  { name: "ナッツ",   role: "WR", animal: "squirrel", fur: FUR.squirrel, stats: "hands" },
  { name: "ゴロ",     role: "OL", animal: "bear",     fur: FUR.bear,     stats: "power" },
  { name: "ガード",   role: "OL", animal: "rhino",    fur: FUR.rhino,    stats: "power" }
];

// 守備チーム（相手）
var DEFENSE = [ ... 5体（role は "DF"） ... ];

// マネージャー（撮影担当。役割・パラメータは持たない）
var MANAGER = { name: "ミケ", fur: 0xf0e2c8, jacket: 0x2f9e8f };
```

`FUR` は毛色のパレット（`FUR.owl` など）。直接 `0xRRGGBB` を書いてもよい。
`TEAM` はユニフォーム色（`home`＝味方/オレンジ、`away`＝相手/青）。
役割は **並び順ではなく `role` フィールド**で決まる（どの位置にどの役割を置いてもよい）。

## 4. 差し替え手順（パターン別）

すべて `characters.js` を編集 → 保存 → ブラウザを再読み込み、で反映されます。

### A. 名前だけ変える
該当キャラの `name` を書き換える。コーチのセリフにも自動反映される。

### B. 色（毛色）を変える
`fur` の値を変える。例：クマを濃い茶色に。
```js
{ name: "ゴロ", animal: "bear", fur: 0x8a5a30 }
```

### C. 動物の種類を変える（見た目だけ）
`animal` を別の種類に変える。役割・パラメータはそのまま。
```js
{ name: "ナッツ", role: "WR", animal: "wolf", fur: 0x9aa7b5, stats: "hands" }
```

### D. 役割を変える
`role` を変える。例：ウサギをパンターに。
```js
{ name: "ピョン太", role: "P", animal: "rabbit", fur: FUR.rabbit, stats: "kicker" }
```

### E. パラメータを変える
`stats` をプリセット名で変える、または直接指定する。
```js
{ name: "ゴロ", role: "OL", animal: "bear", fur: FUR.bear, stats: "speedy" }       // 足の速いクマのライン
{ name: "X",   role: "WR", animal: "cat",  fur: FUR.cat,  stats: { speed:7, power:2, catch:8, kick:1 } }
```

### F. 3要素を自由に組み合わせる（例：足が速い・くま・パンター）
```js
{ name: "ダッシュ", role: "P", animal: "bear", fur: FUR.bear, stats: "speedy" }
```

### G. ユニフォーム色（チーム色）を変える
`TEAM` を変える。
```js
var TEAM = { home: 0xee8e3c, away: 0x3f7fd0 };
```

## 5. 守らなければならない制約（重要）

- **`OFFENSE` と `DEFENSE` はそれぞれ必ず「5体」**にする。
  並べる位置（整列スロット）が5枠固定のため、6体以上にすると表示が崩れる／エラーになる。
- **攻撃チームには `role: "QB"` `"RB"` `"WR"` を1体ずつ入れる**。
  この3役がパス・ランの中心。欠けると先頭の選手で代用されるが、意図通りにならないことがある。
  （`role` で決まるので並び順は自由。位置を入れ替えても役割は変わらない。）
- **色は必ず `0xRRGGBB` の数値**で書く（`"#ee8e3c"` などの文字列は不可）。
- `stats` のプリセット名は実在するものを使う（無い名前を書くと `allround` 扱いになる）。
- 既存の動物の見た目（`makeAnimal` の各分岐）は、必要がなければ変更しない。

---

## 6. 新しい動物の「見た目」を追加する方法（上級）

設定リストだけでは既存6種しか選べません。新種の見た目を足すには `characters.js` の `makeAnimal()` に分岐を1つ追加します。

### パーツの置き方
パーツは `box(幅, 高さ, 奥行, 色, x, y, z)` で1つずつ足します。

- 座標の向き: `y` が上（`0` が地面）、`z+` が前（顔の向き）、`x` は左(−)／右(+)。
- 顔まわりは「頭サブグループ `head`」に追加する（`head.add(box(...))`）。頭の中心はキャラ内で上の方。
- しっぽなど体に付くものは本体 `g` に追加する（`g.add(box(...))`）。
- 完成したキャラは自動で 1.2 倍に拡大される。

### 追加例（テンプレ：ペンギンを足す場合）
```js
} else if (role === "penguin") {
  // 白いおなか
  g.add(box(0.7, 1.0, 0.1, 0xffffff, 0, 1.35, 0.52));
  // くちばし
  head.add(box(0.18, 0.12, 0.18, 0xf4a93c, 0, -0.14, 0.54));
  // 小さな羽
  g.add(box(0.16, 0.7, 0.4, fur, -0.78, 1.35, 0));
  g.add(box(0.16, 0.7, 0.4, fur, 0.78, 1.35, 0));
}
```
追加後は、設定リストで `animal: "penguin"` と書けば使えます。
（`cat` は既に用意済みなので、そのまま `animal: "cat"` で使えます。）

---

## 7. マネージャーの変更

`MANAGER = { name, fur, jacket }` を編集する。
- `fur`: マネージャー（ネコ）の毛色
- `jacket`: スタッフジャケットの色
見た目（カメラを構える等）を変えたいときは `makeManager()` を編集する。

---

## 8. 動作確認

1. `characters.js` を保存する。
2. ブラウザで `index.html` を開く（または再読み込み）。
3. 確認する:
   - 味方5体が意図した動物・色になっているか
   - 相手5体が表示されているか
   - 名前がコーチのセリフに出るか（テーピング発生時など）
   - 画面が真っ白なら、`characters.js` の書き間違い（カンマ抜け・`}` 閉じ忘れ・色を文字列で書いた等）を疑う。ブラウザの開発者ツールのコンソールにエラーが出る。
