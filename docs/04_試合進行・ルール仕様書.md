# 04_試合進行・ルール仕様書

- 目的: ゲーム内の基本ルール、試合進行、得点、能力反映を定義する
- 関連: `05_采配・読み合い仕様書.md`, `10_FGミニゲーム仕様書.md`

## 1. 基本ルール

初心者向けに、まずは攻撃の最重要ルールに絞る。

1. 攻撃側は4回の試行で10ヤード前進できれば、新しい4回のチャンスを得る。
2. 4回以内に10ヤード進めないと、攻撃権が相手に移る。
3. 相手陣のゴールラインまでボールを運べばタッチダウン。
4. タッチダウンは6点。
5. フィールドゴールは成功で3点。
6. 規定シリーズ終了時に、得点が多いチームが勝つ。

## 2. 用語

| 用語 | 意味 |
|---|---|
| ダウン | 攻撃の試行回数 |
| 残りヤード | ファーストダウンまでに必要な距離 |
| ファーストダウン | 10ヤード前進に成功して、攻撃権を継続すること |
| スクリメージライン | プレー開始時にボールが置かれる線 |
| スナップ | プレー開始の球出し |
| ターンオーバー | 攻撃権が相手に移ること |
| タッチダウン | ゴールラインを越えて6点を得ること |
| フィールドゴール | キックでゴールポスト間を通し3点を得ること |
| パント | 得点を狙わず遠くへ蹴り、相手の開始位置を下げること |

## 3. 状態管理

ゲーム状態は最低限、次を持つ。

| 状態 | 内容 |
|---|---|
| `down` | 現在のダウン。1〜4 |
| `toGo` | ファーストダウンまでの残りヤード |
| `los` | スクリメージライン。フィールド上の現在位置 |
| `home` | 味方得点 |
| `away` | 相手得点 |
| `drive` | 現在のシリーズ数 |
| `MAX_DRIVES` | 最大シリーズ数。初期値6 |
| `mode` | `offense` または `defense` |
| `cpuStyle` | 相手攻撃傾向。`run` / `pass` / `balanced` |
| `defLos` | 守備時のフィールド位置 |
| `defPin` | パントなどで相手を押し込んだ補正 |

## 4. 通常プレー

攻撃側は、隊形を選んだあとにプレーを選ぶ。

通常プレー:

- ラン
- ショートパス
- ロングパス

結果は、選択した隊形、プレー、能力値、ランダム要素、相手の読み合いによって決まる。

## 5. ダウン更新

### 5-1. 10ヤード以上進んだ場合

- ファーストダウン
- `down = 1`
- `toGo = 10`
- スクリメージラインを進める

### 5-2. 10ヤード未満の場合

- `down += 1`
- `toGo -= gainedYards`

### 5-3. 4th失敗の場合

- 攻撃権が相手へ移る
- 「ターンオーバー・オン・ダウンズ」として扱う

## 6. 得点

### 6-1. タッチダウン

- ゴールラインを越えるとタッチダウン
- 味方に6点
- 初期実装ではタッチダウン後の追加点は簡略化してもよい
- 将来、エクストラポイントまたは2ポイントコンバージョンを追加可能

### 6-2. フィールドゴール

- 成功で3点
- 失敗すると攻守交代
- 4thダウンの選択肢として出す
- FGミニゲーム導入前は、距離とキック能力による確率判定でよい
- FGミニゲーム導入後は、角度とパワーのプレイヤー操作で成否を決める

### 6-3. セーフティ

- 自陣エンドゾーンで止められると相手2点
- 教育用語としては発生対象
- 初期実装では低頻度・後段対応

## 7. キック

### 7-1. キッカー選定

`role: "P"` の選手がいればその選手をキッカーとして扱う。

いない場合は、`kick` が最大の選手を使う。

### 7-2. フィールドゴール確率式（ミニゲーム導入前）

初期式:

```js
successRate = clamp((1.15 - distance * 0.012) * (kick / 6), 0.1, 0.95)
```

検証例:

- 近距離 `kick=9`: 約95%
- 遠距離 `kick=9`: 約43%
- 近距離 `kick=3`: 約38%

### 7-3. パント

- 得点はない
- 相手を深い位置から始めさせる
- 相手得点率を下げる
- ミニゲーム化しない

## 8. 相手得点・攻守交代

攻守交代のたびに、相手の攻撃を処理する。

現在は次の2方式がある。

### 8-1. 簡易シミュレーション方式

`opponentPossession` により、渡したフィールド位置で相手の得点率が変わる。

結果:

- 0点: 守備で阻止
- 3点: 相手FG
- 7点: 相手TD

検証例:

- 自陣で渡す: 相手得点率が高い
- 敵陣深くで渡す: 相手得点率が下がる
- パント後: 得点率が下がる

### 8-2. 守備采配方式

味方が守備の番になり、守備隊形と戦術を選ぶ。

結果:

- 止める
- FGを許す
- TDを許す
- ターンオーバー奪取

詳細は `05_采配・読み合い仕様書.md` に定義する。

## 9. 能力値の反映

キャラクターの `stats` をプレー結果に反映する。

| 能力 | 影響 |
|---|---|
| `speed` | ランのブレイク、パス後の伸び、見た目の速さ |
| `power` | ランの押し込み、ライン攻防、ファンブルしにくさ |
| `catch` | パス成功率、落球率 |
| `kick` | FG成功率、FGミニゲーム難度、パント距離 |

`6` を平均値として扱う。

## 10. ランの結果

ランでは、RBの `speed` とOLの `power` 平均を使う。

- RBの `speed` が高いほど、大きく抜ける確率と距離が増える
- OLの `power` が高いほど押し込み量が増える
- OLが弱いと止められやすい

検証例:

- `speed=4`: 平均約3.4ヤード
- `speed=9`: 平均約6.0ヤード

## 11. パスの結果

パスでは、WRの `catch` と `speed` を使う。

- `catch` が低いほど落球が増える
- `speed` が高いほどキャッチ後の伸びが増える
- ロングパスは成功時の見返りが大きいが、失敗・インターセプトのリスクが高い

検証例:

- ショートパス `catch=4`: 成功率約66%
- ショートパス `catch=9`: 成功率約79%

## 12. インターセプト

パス失敗時、特にロングパスで相手にボールを取られることがある。

- 発生時は攻守交代
- 用語「インターセプト」を初出解説する
- 将来はリターン演出を追加可能

## 13. ファンブル

ラン、またはキャッチ後のタックル時、低確率でファンブルが発生する。

基本方針:

- 発生率は3〜5%程度
- 走者の `power` が高いほど発生しにくい
- リカバーは乱数で判定する
- 相手が拾えばターンオーバー
- 味方が拾えばその地点で継続

詳細演出は `09_演出・アニメーション仕様書.md` に定義する。

## 14. テーピングによる一時補正

テーピング中の選手がいる場合、その選手の担当プレーが次の1プレーのみ少し不利になる。

- RB/OL: ランの獲得ヤード減
- QB/WR: パス成功率・獲得ヤード減
- ロングパスはインターセプト率微増

詳細は `06_マネージャー業務仕様書.md` に定義する。

## 15. 試合終了

- 自分の攻撃が一区切りするごとにシリーズを1消化する
- 初期値は `MAX_DRIVES = 6`
- 規定シリーズに達したら試合終了
- スコアで勝ち、負け、引き分けを表示する
- 「もう一度あそぶ」で再戦できる

HUDには、例として「シリーズ 3/6」を表示する。

## 16. 初期値・調整値

| 項目 | 初期値・式 | 備考 |
|---|---|---|
| 最大シリーズ数 | `MAX_DRIVES = 6` | 試合の長さ |
| 平均能力 | `6` | `stats` の基準値 |
| FG成功式 | `(1.15 - 距離*0.012) * (kick/6)` | 0.1〜0.95にクランプ |
| ファンブル率 | 3〜5% | 低頻度 |
| テーピング効果 | 次の1プレー | 軽い不利のみ |

## 17. 制約・非対象

- 本格的な時計管理は初期版では導入しない
- クォーター制は後段
- 複雑な反則の受諾/拒否は自動化する
- 公式ルールの細部より、初心者が試合を理解することを優先する
- 相手オフェンスの全プレー可視化は後段
