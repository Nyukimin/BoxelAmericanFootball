# 仕様追記：記録・分析マネージャー（撮影キャラクター）

- 作成日: 2026年6月14日
- 位置づけ: 企画書「ボクセル・フットボール（仮称）」への追記
- 関連章: 第3章（プレイヤーの役割：監督）／第6章（プレー解説リプレイ）／第7章（アート方針・データ駆動）

## 1. 目的・背景

企画書のサポート業務分析（記録・分析）を、ゲーム画面上で可視化する。対象ユーザーである「初心者のマネージャー」が担う代表的な実務が「試合の撮影・記録」であることを、サイドラインに撮影キャラクターを置くことで自然に伝える。第6章「プレー解説リプレイ」の“映像の出どころ”を象徴する存在でもある。

## 2. 仕様：撮影マネージャーの配置

- 役割: フィールド外のサイドラインに立ち、試合を撮影し続ける記録・分析担当のマネージャー。
- 操作対象ではない（プレイヤーは監督として采配に集中する。第3章の役割は変更しない）。
- 見た目: ぽっちゃり動物（ネコ風）のボクセルキャラクター。スタッフ用ジャケット姿でビデオカメラを顔の前に構える。カメラには赤い録画ランプ（REC）を付ける（第7章のアート方針に準拠）。
- 配置: フィールド外側のサイドライン付近、ほぼ中央。プレーの邪魔をしない位置。
- 挙動:
  - 常にボールの方を向いてパンする（撮影している演出）。
  - 録画ランプが点滅する。
  - 軽いアイドル動作（わずかな上下の揺れ）。
- 非対象: ゲームのルール・采配・判定・選手アニメーションには一切干渉しない純粋な演出。テンポを損なわない（第6章「任意だからこそ続く」の方針に合わせ、プレイの流れを止めない）。

## 3. 既存仕様との関係

- 第3章（コアループ＝采配）: 変更なし。マネージャーは采配の対象ではない。
- 第6章（プレー解説リプレイ）: 本キャラクターはリプレイ＝記録・分析の象徴。将来、撮影した映像からリプレイに入る導線として拡張できる。
- 第7章（データ駆動のキャラ設計）: マネージャーも「1キャラ＝1設定」の枠組みに将来組み込み可能（名前・動物・チームカラーの差し替え対象）。現状は仮メンバー扱い。

## 4. 実装状況

- `index.html` にプロトタイプとして実装済み。
  - `makeManager()`: ボクセルのネコ風マネージャー＋ビデオカメラ（本体・レンズ・RECランプ）を生成。
  - サイドラインに配置し、毎フレーム「ボール追従パン」「RECランプ点滅」「アイドル動作」を更新。
- ゲームロジックには非接続（装飾レイヤーのみ）。

## 5. 今後の拡張余地（任意・後段）

- スタッツ表示: 撮影マネージャーの記録として、ヤード・成功率などの簡易スタッツをHUDに表示。
- 分析の采配還元: 「相手はランを警戒」などの傾向を提示し、次の隊形・プレー選択のヒントにする（第6章の学習設計と連動）。
- リプレイ導線: 撮影キャラからプレー解説リプレイへ入る入口を設ける。

いずれも企画書の方針（初心者向けにルールを絞る／テンポ優先／学習を強制しない）に沿って、まずは「動く試作で手応えを確かめてから」後段で追加する。

## 6. 制約・非対象（明示）

- 本追記は企画書にない新規要素のため、原本（企画書.docx）は変更せず、本ドキュメントを追記として扱う。
- 見た目（カメラの構え・向き・視認性）はWebGL実機での確認が必要。位置・スケールは調整余地あり。
