# 06_マネージャー業務仕様書

- 目的: マネージャー教材として、記録・撮影・分析・選手ケアをゲーム内で扱う方法を定義する
- 上位仕様: `01_教材コンセプト仕様書.md`

## 1. 位置づけ

本仕様は、ゲームを「マネージャー用教材」として成立させる中核である。

マネージャー業務を単なる飾りにせず、試合理解・記録・分析・選手ケアの入口として扱う。

## 2. 対象業務

| 業務 | ゲーム内表現 | 教材としての意味 |
|---|---|---|
| 試合撮影 | サイドラインの撮影マネージャー | プレー記録・リプレイの出どころを理解する |
| 記録 | プレー後の記録表示 | ダウン、ヤード、結果を読めるようにする |
| 分析 | 「見るポイント」「次の判断」 | なぜ進んだか、なぜ止まったかを考える |
| 選手ケア | テーピング補助 | コンディション確認の重要性を知る |
| サイドライン視点 | フィールド外のキャラ配置 | プレイヤーが選手でなく支える立場だと分かる |
| 用語メモ | 用語図鑑との連携 | 現場用語の復習に使う |

## 3. 記録・分析マネージャー

### 3-1. 目的

対象ユーザーである初心者マネージャーが担う代表的な実務「試合の撮影・記録」を、サイドラインに撮影キャラクターを置くことで自然に伝える。

また、第6章相当の「プレー解説リプレイ」における映像の出どころを象徴する存在にする。

### 3-2. 役割

- フィールド外のサイドラインに立つ
- 試合を撮影し続ける
- 記録・分析担当のマネージャーとして見える
- 操作対象ではない
- プレイヤーは監督として采配に集中する
- ゲームのルール・判定・選手アニメーションには干渉しない

### 3-3. 見た目

- ぽっちゃり動物
- ネコ風
- スタッフ用ジャケット姿
- ビデオカメラを顔の前に構える
- カメラには赤い録画ランプを付ける
- ボクセル表現
- 可愛く、画面の邪魔をしない

### 3-4. 配置

- フィールド外側のサイドライン付近
- ほぼ中央
- プレーの邪魔をしない位置

### 3-5. 挙動

- 常にボールの方を向いてパンする
- 録画ランプが点滅する
- 軽いアイドル動作をする
- わずかな上下揺れで生きている感じを出す

### 3-6. 実装状況

実装済み扱い。

- `makeManager()`: ボクセルのネコ風マネージャーとビデオカメラを生成
- サイドライン配置
- ボール追従パン
- RECランプ点滅
- アイドル動作
- ゲームロジックには非接続

## 4. プレー後の記録表示

撮影マネージャーと連動し、プレー後に短い記録を表示する。

### 4-1. 表示項目

| 項目 | 例 |
|---|---|
| プレイ種別 | ラン / ショートパス / ロングパス / FG / パント |
| 隊形 | ショットガン / アイ / シングルバック / ゴールライン |
| 結果 | +4yd / インコンプリート / インターセプト |
| 次の状況 | 2nd & 6 |
| 得点変化 | TD +6 / FG +3 |
| 用語 | ファーストダウン / ターンオーバー |
| 見るポイント | ラインが押した / 守備が中央に寄った |

### 4-2. 表示例

```text
記録: ラン +4yd
状況: 2nd & 6
見るポイント: 中央のラインが少し押し込んだ
次の判断: 残り6yd。ランもパスも選べる場面
```

### 4-3. 方針

- 長文にしない
- 1プレーの流れを止めない
- 任意でリプレイや詳細へ進める
- 初心者が記録の書き方を真似できる表現にする

## 5. 分析の采配還元

後段では、撮影マネージャーが分析ヒントを出す。

例:

- 「相手はランを警戒して中央に寄っているみたい。」
- 「さっきから短いパスが通っているよ。」
- 「残り2ヤードだから、確実に進めたい場面。」
- 「ゴール前だから、密集隊形もあり。」

これは答えを教えるものではなく、判断材料を増やすものにする。

## 6. テーピング補助

### 6-1. 目的

マネージャーのサポート業務のひとつである「けが・テーピング補助」を、ゲーム画面上に低頻度の演出として可視化する。

アメフトでは接触があるが、本作では痛々しく見せず、選手をケアして送り出すマネージャーの役割を可愛く伝える。

また、テーピング中の選手の担当プレーを一時的に不利にすることで、選手のコンディションも采配の判断材料になることを学べるようにする。

### 6-2. 発生タイミング

攻撃が継続する穏やかな場面にのみ発生させる。

発生対象:

- ファーストダウン獲得後
- 通常前進後
- 次のプレー選択前

発生させない場面:

- ターンオーバー
- タッチダウン
- 攻守交代
- その他慌ただしい場面

### 6-3. 頻度制御

| 項目 | 初期値 | 意味 |
|---|---:|---|
| 発生確率 | 15% | 攻撃継続時に判定 |
| クールダウン | 3プレー | 一度発生した後の再発生禁止期間 |
| 効果持続 | 1プレー | 次の1プレーだけ不利 |

追加制約:

- 既にテーピング中の選手には重ねない
- 多発させない
- 初心者のテンポを壊さない

### 6-4. 演出内容

- 攻撃側の動物選手1体に白いテーピング帯を付ける
- ボクセルの帯で表現する
- マスコットコーチが一言フォローする
- 不利になるプレーを伝える

例:

```text
ピョン太をテーピング中。今回はランがちょっと不利かも。ほかの手もあるよ。
```

### 6-5. ゲーム性への影響

次の1プレーのみ、軽く不利にする。

| 対象役割 | 不利になるプレー | 影響 |
|---|---|---|
| RB | ラン | 獲得ヤード減 |
| OL | ラン | 獲得ヤード減 |
| QB | パス | 成功率・獲得ヤード減 |
| WR | パス | 成功率・獲得ヤード減 |
| QB/WR | ロングパス | インターセプト率微増 |

初期調整値:

| 補正 | 初期値 |
|---|---:|
| ラン不利 | 獲得ヤード × 0.55 |
| パス不利 | 失敗率上昇 + 獲得ヤード × 0.7 |

### 6-6. 回復

- 効果は1プレーで終わる
- プレー後にテーピングが外れる
- 通常状態に戻る
- 長期離脱や退場は行わない

### 6-7. 実装状況

実装済み扱い。

- `maybeTape()`: 攻撃継続時に確率・クールダウン判定し、ランダムな攻撃選手に白いテーピング帯を付与
- `outcome()`: 対応プレーに軽い不利補正
- `recoverTape()`: `finalize()` 先頭で呼び、1プレーで回復
- `chooseFormationUI()`: 不利プレーを伝えるコーチの一言を表示

## 7. マネージャー業務とUIの接続

### 7-1. サイドラインメモ

プレー後に、マネージャー視点のメモを出す。

例:

```text
マネージャーメモ:
3rd & 2。次はファーストダウンまであと少し。
ランなら確実、パスなら裏をかけるかも。
```

### 7-2. 用語メモ

用語初出時、用語図鑑へ記録する。

例:

```text
用語メモに追加: パント
```

### 7-3. 撮影からリプレイへ

撮影マネージャーを、プレー解説リプレイの導線にする。

例:

- カメラアイコン
- 「今のプレーを見返す」ボタン
- 撮影マネージャーの吹き出し

## 8. 今後の拡張

- スタッツ表示: ヤード、成功率などの簡易スタッツをHUDに表示
- 分析ヒント: 相手傾向を提示し、次の隊形・プレー選択のヒントにする
- リプレイ導線: 撮影キャラからリプレイへ入る
- コンディション管理: 疲労がたまるとテーピング頻度が上がる
- 軽いミニ操作: ボタンで手当てして復帰させる

## 9. 制約・非対象

- マネージャーキャラは操作対象にしない
- ゲームのルール・判定に直接干渉しない
- テーピングは攻撃側のみ
- 守備側・相手チームのテーピングは対象外
- 選手の退場、長期離脱、強いペナルティは行わない
- 痛々しい表現はしない
- 原本企画書のコアループを壊さない
