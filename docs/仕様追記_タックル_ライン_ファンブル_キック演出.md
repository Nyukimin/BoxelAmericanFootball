# 仕様追記：タックル／ラインの攻防／ファンブル／キック演出

- 作成日: 2026年6月14日
- 状態: 設計（未実装）。本ドキュメントは実装前の仕様。
- 位置づけ: 企画書「ボクセル・フットボール（仮称）」への追記
- 関連章: 第4章（采配システム）／第5章（視点）／第7章（アート方針：痛々しくない）／第9章（進行）

## 0. 用語の整理（重要）

- 「スクラム」は**ラグビー**の用語で、アメフトには存在しない。アメフトの相当物は次の3つ:
  - スクリメージライン（ボールを置く線）／スナップ（球出し）／**ラインの攻防（ブロック）**
- 本書の「ラインの攻防」は、上記のアメフトのライン同士の押し合いを指す。

現状（実装済み）では、タックルもライン攻防も「数値だけ」（`outcome()` の `power` 判定）で、画面上の動作は無い。本書はそれを可視化し、ファンブルとキック演出を足す設計。

---

## 1. 視覚的タックル

### 目的
ラン／パスの終わりに、守備が当たって止める動作を見せる（数値タックルの可視化）。

### 挙動
- プレー終了地点（`endZ`）付近の最寄り守備選手1体が走者に寄って衝突し、走者がポヨンと止まる（痛々しくしない／第7章準拠）。
- 走者の `speed` が高い・守備が弱いと、止まる位置が少し先になる（振り切りの表現）。

### stats連動
- 実ヤードは `outcome()` で確定済み。タックル位置・タイミングは結果に合わせる（見た目）。

### 実装の勘所
- `updateAnim()` のラン／パス終端で、最寄り `defense` を `carrier` 方向に寄せ、接触時に小さなノックバック（一瞬のけぞる）。
- 既存のアニメ完了タイミング（`finalize` 呼び出し）と整合させる。

### 制約
- 結果ヤードは変えない純粋演出（ファンブルと連動する場合のみ例外＝下記）。

---

## 2. ラインの攻防（スナップ時）

### 目的
スナップ直後に味方ライン（OL）と守備前列（DL）が押し合う動作を見せ、采配と `power` が見た目で伝わるようにする。

### 挙動
- スナップで両ラインが1〜2歩ぶつかる。`power` が高い側が押し込む。
- ランなら走路ができる／塞がる様子を見せる。

### stats連動
- OLの `power` 平均 対 守備で、押し勝ち／押し負けの見た目を変える。

### 実装の勘所
- `startPlay()` 冒頭に短い「ライン押し合い」段を追加（`phase: "snap"` を新設、または anim の前段）。
- 既存のボール挙動の前に挟むため、タイミング競合に注意。

### 制約
- 演出中心。結果は `outcome()` で確定。

## 3. ファンブル（新ルール）

### 目的
タックル時にまれにボールを落とし、拾い合いで攻守が変わる新リスクを足す（INTに次ぐターンオーバー要素）。

### 挙動
- ラン、またはキャッチ後のタックル時、低確率でファンブル発生。
- ボールがこぼれる短い演出 → リカバー（どちらが拾うか）を確率判定。
- 相手が拾えばターンオーバー（`endPossession` へ）。味方が拾えばその地点で継続。

### stats連動
- 走者の `power`（または落ち着き）が高いほどファンブルしにくい。強いタックルほど発生しやすい。

### ゲーム性・調整
- 多発させない（テーピング同様、低頻度＋クールダウンを検討）。
- 調整値（初期案）: 発生確率 3〜5%／相手リカバー率 50%。

### 物理演算の扱い（方針・決定事項）
- **結果（誰が拾うか）は乱数で決める**（`power` や状況で重み付け）。リアル物理で判定しない。
  - 理由: 物理任せはバランス調整が困難で理不尽になりやすい。実際のアメフトでもリカバーは運の要素が大きく、ほぼ全てのアメフトゲームが乱数処理。本作の初心者向け・テンポ重視にも合致。
- **ボールの跳ねは簡易擬似バウンドで表現**（バウンドのたびに方向へ乱数＋減衰）。剛体物理エンジンは使わない。
  - 理由: 楕円（回転体）の厳密な接地・回転計算は過剰で重い。単一HTML・依存なしの方針に反する。既存のパスも擬似放物線であり、それと統一する。楕円の「予測不能な跳ね」は乱数方向のバウンドで十分“それっぽく”出せる。
- **物理エンジンは非採用**（追加ライブラリなし）。

### 実装の勘所
- `outcome()` に `fumble` フラグを追加、または `finalize()` 直前で判定。
- ターンオーバー時は既存の `endPossession()` を利用。

### 制約
- 痛々しくしない。低頻度に抑える。

## 4. キックの演出

### 目的
現在は数値だけのフィールドゴール／パントに、ボールを蹴って放物線で飛ぶアニメを足す。

### 挙動
- キッカーがボールを蹴る → 放物線で飛ぶ。
- フィールドゴールはゴールポスト（H型）へ。成功／失敗で軌道が変わる（結果に合わせる）。
- パントは大きく飛んで相手陣へ。

### 実装の勘所
- 既存のパスの放物線（`updateAnim` のパス処理）を流用。
- `kickPlay()` にアニメ段を追加し、`ok`（成否）に応じて軌道を分岐。
- `field()` にゴールポストのオブジェクトを足すと分かりやすい。

### 制約
- 成否は `kick` 力・距離で確定済み。演出は結果に合わせるだけ。

---

## 5. 実装順の推奨

1. キックの演出（既存の放物線流用で低コスト・効果大）
2. 視覚的タックル（プレーの締まりが出る）
3. ラインの攻防（采配の手応えが増す）
4. ファンブル（新ルール。バランス調整が要るので最後）

## 6. 付録：他のアメフトゲームの定番要素（今後の候補）

代表的なアメフトゲーム（Madden / Retro Bowl / Tecmo Bowl / NFL Blitz など）でよく見る要素。本作の方針（初心者向け・可愛い・テンポ重視）に照らした適性つき。

| 要素 | 内容 | 本作への適性 |
| --- | --- | --- |
| 2ポイントコンバージョン | TD後に7点狙わず2点を狙う選択 | 高（采配の択が増える・実装軽い） |
| エクストラポイント | TD後のキック（現状は自動7点） | 中（キック演出と相性） |
| キック/パントリターン | 蹴られた球を runner が走って返す | 中（スペシャルチームの華・実装やや重） |
| QBスクランブル | パスが無いとQB自身が走る | 中（プレーの幅・実装中） |
| オーディブル | スナップ前に作戦変更 | 中（上級者向け・初心者には任意化） |
| プレイアクション | ランに見せかけてパス | 中（采配の駆け引き） |
| 天候（雨/雪/風） | キック成功率や落球率が変化 | 高（既存statに掛けるだけ・可愛く表現可） |
| スタミナ/疲労 | 連続出場で能力低下→交代 | 中（テーピングと連動可） |
| スター選手の必殺 | 特定選手が決め所で能力UP | 高（可愛い演出と相性） |
| シーズン/育成モード | 試合を重ねて選手が成長・編成 | 高（データ駆動の3要素と好相性／規模大） |
| ペナルティ（反則） | オフサイド等で罰退 | 低（初心者には複雑・テンポ阻害） |
| セーフティ（2点） | 自陣エンドゾーンで止められる | 低（稀ケース・理解が難しい） |
| 2ミニッツ/クロック管理 | 時間との駆け引き | 中（クォーター制を入れるなら） |
| ターンオーバーリターン | INT/ファンブルを拾って走る | 中（ファンブルと一緒なら自然） |

### おすすめの次の一手（軽くて効果が大きい）
- 2ポイントコンバージョン（采配の択・実装軽い）
- 天候（既存statに係数を掛けるだけで駆け引きが増える）
- スター選手の必殺（可愛さとゲーム性の両立）

シーズン/育成モードは魅力的だが規模が大きいので、上記が固まってから検討するのが無難。
