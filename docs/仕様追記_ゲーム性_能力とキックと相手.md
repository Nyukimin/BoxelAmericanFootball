# 仕様追記：ゲーム性（能力・キック・相手の得点・試合終了）

- 作成日: 2026年6月14日
- 位置づけ: 企画書「ボクセル・フットボール（仮称）」への追記
- 関連章: 第4章（采配システム）／第9章（進行と勝利条件）／選手の3要素（role/animal/stats）

## 1. 目的

「能力が一定・落球しない・キックを外さない」でゲーム性が薄かった状態を改善し、アメフトとして成立する最低限の駆け引きを入れる。選手の `stats` と `role`、フィールド位置が結果に効くようにする。

## 2. 能力（stats）が結果に効く

`stats`（speed/power/catch/kick、6が平均）をプレー結果へ反映する（`outcome()`）。

- ラン: RBの `speed` でブレイク（大きく抜ける）確率と距離が増える。OL（ライン）の `power` 平均で押し込み量が増え、弱いと止められやすい。
- パス: WRの `catch` が低いほど**落球**が起きる（成功率が下がる）。`speed` で受けた後の伸び（YAC）が増える。
- 検証例: ラン平均 speed4=3.4→speed9=6.0ヤード／ショート成功率 catch4=66%→catch9=79%。

## 3. キック（4thダウンの選択肢）

4thダウンで「フィールドゴール」「パント」を選べる。成否はキッカー（`role:"P"` がいればその選手、いなければ `kick` 最大の選手）の `kick` 力と距離で決まる。

- フィールドゴール: 成功で3点。近いほど・kick力が高いほど決まる。遠いと外す。
  - 検証例: 近距離kick9=95%／遠距離kick9=43%／近距離kick3=38%。
- パント: 得点はないが相手を深い位置に押し込み、相手の得点を抑える。

## 4. 相手の得点（攻守交代）

攻守交代（インターセプト・ダウン更新失敗・パント・FG失敗・自分のTD/FG後のキックオフ）のたびに、相手の攻撃を簡易シミュレートする（`opponentPossession`）。

- 渡したフィールド位置で相手の得点率が変わる。自陣で渡すほど相手は得点しやすく、敵陣深くで渡す（またはパントで押し込む）ほど抑えられる。
- 結果は 0点（守備で阻止）／3点（FG）／7点（TD）。
- 検証例: 自陣交代70%・敵陣交代37%・パント後52%（得点される確率）。

## 5. 試合の進行と終了

- 自分の攻撃が一区切り（得点・攻守交代）するごとに「シリーズ」を1消化する。
- 規定シリーズ（初期値 `MAX_DRIVES = 6`）で試合終了。スコアで勝ち／負け／引き分けを表示し、「もう一度あそぶ」で再戦できる。
- HUDのヤード表示に現在のシリーズ数（例「シリーズ 3/6」）を表示。

## 6. 調整パラメータ（実装上の初期値）

| 項目 | 場所 | 備考 |
| --- | --- | --- |
| シリーズ数 | `MAX_DRIVES = 6` | 試合の長さ |
| FG成功式 | `kickPlay` | `(1.15 - 距離*0.012) * (kick/6)` を0.1〜0.95でクランプ |
| 相手得点率 | `opponentPossession` | フィールド位置 `goodness` で TD/FG 確率が変動 |
| 能力係数 | `outcome` | `stat/6` を係数化（6が平均=1.0） |

いずれも体感を見て調整可能。検証スクリプトは `docs/調査/` の方式で再現できる。

## 7. 制約・非対象（明示）

- 相手の攻撃は数値シミュレーション（画面上で相手オフェンスは動かさない）。
- クォーター制・時間（クロック）は未導入。区切りは「シリーズ数」で簡易化。
- `power/catch/kick` のうち、さらに細かい連動（ラインの個別対面、QB精度など）は今後の拡張。
