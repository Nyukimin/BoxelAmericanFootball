- 作成日: 2026年6月14日
- 状態: 再編版ドラフト

## 1. 最上位方針

本作は、初心者マネージャーがアメフトの試合を見て、状況・用語・記録・選手状態を理解できるようになるための教材である。

ゲーム性は教材性を支える中核であり、削ぎ落とす対象ではない。プレイヤーが「理解したから判断できた」「判断できたから勝てた」と感じられる状態を目指す。

副次的に、新入部員がアメフトの基本ルール、ポジション、作戦、試合の流れを学ぶ入門教材としても使えるものにする。

## 6. 実装状態の扱い

この再編版は新しい仕様書群として作成する。元の企画書 `.docx` と旧追記仕様は変更しない。旧文書は統合元として残し、今後の正本はこの仕様書群へ移す。


各仕様書では、以下の状態を明示する。

| 状態 | 意味 |
|---|---|
| 実装済み | 既に `index.html` 等へ組み込まれている。必要に応じて目視確認が残る |
| 設計済み・未実装 | 仕様はあるが実装前。TDDで進める |
| 後段検討 | 方向性だけあり、採用・詳細化は未決定 |
| 非対象 | この版では扱わない |

## 7. 優先順位

最上位優先順位は次の通り。

1. マネージャー教材として必要な理解を損なわない
2. ゲームとしての読み合い・手応え・テンポを損なわない
3. 初心者が混乱しないよう、用語とルールは必要なときだけ短く出す
4. 可愛く、痛々しくない見せ方にする
5. 仕様と実装の差分をテストで管理する

## 8. 次に優先する実装候補

現状の未実装重要項目は次の順で扱う。

1. 1プレーの可視化の基盤
2. キック演出
3. 視覚的タックル
4. ラインの攻防
5. ファンブル
6. FGミニゲーム
7. 用語図鑑
8. 反則システム

ただし、教材価値を最短で上げる場合は、プレー後の「記録」と「見るべきポイント」表示を先に入れる。

## 9. 命名規則

- 仕様書は `NN_名称仕様書.md` 形式とする。
- 実装対象の関数名、ファイル名、状態名はバッククォートで囲む。
- 用語は初出時に、可能な限り日本語説明を添える。
- アメフト用語を誤用しない。特に「スクラム」はラグビー用語であり、アメフトでは「スクリメージライン」「スナップ」「ラインの攻防」を使う。
