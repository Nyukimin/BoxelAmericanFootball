## 順序の意味（不変な遷移）
- 攻撃: 編成 → 隊形 →（守備テル）→ プレー →[ターゲット | FG2タップ | 無] → 結果 → タップ送り → 次ダウン
- 4thダウンのみ FG/パントが追加。TD後は PAT/2pt → キックオフ（自得点はオンサイド可）。
- 守備: 構え → 作戦 を相手の各ダウンで繰り返し。
- 攻守交代の契機: ダウン切れ / インターセプト / ファンブル喪失 / 得点 / セーフティ / パント / FG。
- 試合長はクロック（Q1-4、同点でOT=Q5）。

## E2E が検証する invariant（tests/e2e/user_sim.mjs）
- 例外なし / 進行不能スタックなし / 必ず gameOver 到達
- home,away >= 0 / 得点増分 ∈ {0,1,2,3,6,7,8}
- 攻撃中 down ∈ 1..4 / 守備中 oppDown ∈ 1..5
- quarter ∈ 1..5 / 同一クォーター内で clock 単調減少
- los ∈ [0,120]
- 味方は手前(低z)・相手は奥(高z)（攻撃の意思決定点でユニットの平均zが反転しない）

## 結果（seed=1..10）
全10シード invariant 緑（試合終了到達・スコア妥当・各不変条件満たす）。自動シムでは不具合を検出せず。
