## B. 演出・アニメーション（09章）

- 視覚的タックル（守備が当たって止める）: 実装済み。プレー終了時、ゴール未到達なら最寄り守備がキャリアへ突っ込んで当たり→キャリアが後ろ向きに転倒（倒れる）。ゴール到達（TD）は倒れず喜ぶジャンプ。傾き(rotation)と位置 y のみで表現し、結果ヤード（ball.position.z）は不変
- ラインの攻防（スナップ時の押し合い）: 実装済み（`lineClashShift`。DL/OL が power 差でシフト）
- 全選手が動く（役割づけ）: 実装済み。攻撃時、OL=最寄りDLに当たってブロック（power差で前後）、QB=ドロップバック/フェイク、非ターゲットのWR/RB=斜め前へルート、DL=最寄りOLに当たって押し合う、守備のLB/CB=キャリア追走。棒立ちを解消（`nearestIn` で対戦相手を選ぶ）
- 敵守備の役割分担: 実装済み。DL(ライン担当)×2 / LB(中盤で追走)×2 / CB(俊足で追走)×1。DL は追走せずライン攻防に専念、LB/CB は追走（CB が最速）
- 読み合い（守備カバレッジ）: 実装済み。攻撃時、隊形決定後に守備が blitz/man/zone を選び配置でテルを見せる。`coverageMatchup` の三角関係（short>blitz / long>man / run>zone）で `outcome()` の成功率・ヤードを補正し、メッセージで読みの成否を説明。coverage 未指定は従来挙動（後方互換）
- 相手攻撃の4ダウン制（攻守対称）: 実装済み。守備時は相手も `oppLos/oppDown/oppToGo` を持ち1プレーずつ前進。毎ダウン、味方の守り作戦（マン/ゾーン/ブリッツ）を選び、CPUは `pickCpuPlay` でラン/ショート/ロングを選択、`outcome()` を流用して解決。10ヤードで相手ファーストダウン / 4回守れば味方ボール / TD・インターセプトで決着。各プレーを `startDefensePlay`/`updateDefenseAnim` で可視化
- メッセージ送り: 実装済み。プレー結果・攻守交代・相手ドライブ後は「▶ タップで次へ」クリックまで待つ（30秒の安全フォールバック）。`waitContinue`
- 注: `defenseDrive`（ドライブ一括シミュレーション）は上記の毎ダウン解決へ置換され、現フローでは未使用（純粋関数とテストは将来の自動シミュレーション用に保持）
- キック演出（放物線＋H型ゴールポストを `field()` に追加）: 未実装
- ファンブルの発火＋擬似バウンド演出: 未実装
- ミニマップ（ボール位置）: 未実装
- プレー解説リプレイUI（もう一度見る／スキップ／役割ラベル）: 未実装
