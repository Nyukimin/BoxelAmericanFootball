# 14_実装仕様：1プレーの可視化（全選手が動く因果）

- 目的: 1プレー中に全選手が動き、因果（スナップ→ライン攻防→走路→走/投→守備反応→タックル）が画面で見えるようにする
- 状態: 実装仕様（TDD→ヘッドレス→修正で実装）
- 関連: `09_演出・アニメーション仕様書.md`（方針）, `04_試合進行・ルール仕様書.md`（結果は確定済み）
- 背景: 実プレイで「キックして受けるだけ／相手も味方も動かない」と見える欠落を解消する。結果ヤードは `outcome()` で確定済みで、本仕様は見た目のみ。

## 1. アニメフェーズ（updateAnim を拡張）

既存の run/pass アニメに、全選手の動きを重ねる。

```
snap -> line_contact -> run/pass -> defense_react -> tackle -> finalize
```

- snap: 既存（ボール出し）
- line_contact: OL と DL が LOS 付近でぶつかり、押し勝った側へ少しシフト
- run/pass: キャリア（rb/wr）の既存パス
- defense_react: 全守備がボール/キャリアへ寄る（追走）
- tackle: 最寄り守備がキャリアへ収束し、接触でキャリアがポヨンと停止
- finalize: 既存

## 2. 動かす対象（全10選手）

| 対象 | 動き |
|---|---|
| OL（攻撃ライン idx 3,4 等の非キャリア） | スナップで前進し LOS で押し合い。押し勝ち/負けでシフト量が変わる |
| キャリア（run=rb / pass=wr） | 既存の走路/受球パス（endZ まで） |
| QB | run=ハンドオフ後に少し下がる / pass=軽くドロップバック |
| 守備ライン | OL と押し合い（反対向き） |
| 守備（全体） | キャリアへ追走（pursuit）。最寄り1体が収束＝タックル |

## 3. 純粋ロジック（gamelogic.js・TDD対象）

副作用なしの小関数を追加（既存関数は変更しない）。

```js
// 最寄り守備のインデックス（タックラー選定）
nearestDefenderIndex(carrier, defenders) -> number
//   carrier = {x, z}, defenders = [{x, z}, ...]。距離最小の index。

// ライン攻防のシフト量（符号つき）。攻撃が押し勝てば +、負ければ -
lineClashShift(olPower, dlPower, maxShift) -> number
//   例: clamp((olPower - dlPower)/6, -1, 1) * maxShift

// 追走の補間ターゲット（0..1 の進行 t で守備をキャリアへ寄せる）
pursuitTarget(defender, carrier, t) -> {x, z}
//   lerp(defender, carrier, clamp(t,0,1))
```

## 4. 統合（index.html）

- `startPlay()` 開始時に、守備の初期位置を `userData.dz0` 等に退避。
- `updateAnim(dt)` のラン/パス処理に、毎フレーム以下を追加:
  - 進行度 `p` に応じて OL/DL を `lineClashShift` 分だけ押し合いシフト。
  - 全守備を `pursuitTarget` でキャリアへ寄せる（弱めの係数）。
  - `nearestDefenderIndex(キャリアの最終付近, 守備)` で選んだタックラーは、終盤 `p>0.8` でキャリアへ強く収束。
  - 接触時（`p>=1` 付近）キャリアを `y` 方向にポヨン（既存の sin 揺れを流用）。
- 既存の `finalize()` 呼び出しタイミングは変えない。結果ヤードも変えない。

## 5. ヘッドレス検証（tests/headless/flow.mjs 拡張）

ラン1プレーを駆動し、以下を assert（モックの position.x/z を読む）:
- (k) プレー中に守備の少なくとも1体の位置が動く（z が初期から変化）。
- (l) プレー中に非キャリアの攻撃選手の少なくとも1体が動く（ライン攻防）。
- (m) 終了時、選ばれたタックラーがキャリアの最終位置の近く（距離がしきい値以下）にいる。
- 既存 (a)-(j) は非回帰で全て通す。例外なし。

## 6. 制約

- 見た目のみ。`outcome()` の結果ヤード・成否は変えない。
- 痛々しくしない（ポヨン）。アニメ時間を伸ばしすぎない。
- 初期版は全11人の本格AIではなく、上記の簡易振り付け。
- 相手オフェンス（味方が守備のとき）の全可視化は後段（本仕様は味方攻撃時の可視化が対象）。
