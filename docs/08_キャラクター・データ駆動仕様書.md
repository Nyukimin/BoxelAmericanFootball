# 08_キャラクター・データ駆動仕様書

- 目的: 選手・マネージャーの差し替え、役割、見た目、能力値を定義する
- 関連: `06_マネージャー業務仕様書.md`, `09_演出・アニメーション仕様書.md`

## 1. 基本方針

キャラクターは「データ」と「見た目」を分けて管理する。

ほとんどの差し替えは `characters.js` の設定リストを書き換えるだけで完了し、ゲーム本体である `index.html` は原則触らない。

## 2. ファイル構成

| ファイル | 役割 | 編集方針 |
|---|---|---|
| `characters.js` | キャラ定義、設定リスト、見た目ビルダー | キャラ差し替え時に編集する |
| `index.html` | ゲーム本体、ルール、采配、描画 | 原則触らない |
| `gamelogic.js` | 純粋ロジック。後段抽出対象 | 能力値反映などをテスト可能にする |

`characters.js` は読み込まれると `window.BoxelChars` を公開し、`index.html` がそれを使ってキャラを並べる。

## 3. 1選手の構成

1選手は、次の3要素でできている。

```text
役割 role + 見た目 animal/fur + パラメータ stats
```

例:

```js
{ name: "ダッシュ", role: "P", animal: "bear", fur: FUR.bear, stats: "speedy" }
```

| 項目 | 意味 | 例 |
|---|---|---|
| `name` | 表示名。コーチのセリフ等で使う | `"ミミィ"` |
| `role` | 役割、ポジション | `"QB"` |
| `animal` | 見た目の動物 | `"owl"` |
| `fur` | 毛色。`0xRRGGBB` 形式 | `0xe7d4a8` |
| `stats` | 能力プリセット名、または直接オブジェクト | `"speedy"` |

## 4. 役割 `role`

| 値 | 役割 | 教材上の説明 |
|---|---|---|
| `QB` | クォーターバック | 司令塔。パスを投げる |
| `RB` | ランニングバック | 走る役 |
| `WR` | レシーバー | パスを受ける役 |
| `OL` | ラインマン | QBを守る、走路を作る |
| `P` | パンター/キッカー | 蹴る役 |
| `DF` | ディフェンス | 守備チーム用 |

## 5. 能力値 `stats`

能力値は、`speed`、`power`、`catch`、`kick` を1〜10で持つ。6を平均値として扱う。

| 能力 | 意味 | ゲーム内影響 |
|---|---|---|
| `speed` | 足の速さ | ランの伸び、YAC、見た目の速さ |
| `power` | パワー | ライン攻防、ラン押し込み、ファンブル耐性 |
| `catch` | 捕球 | パス成功率、落球率 |
| `kick` | キック | FG、パント、FGミニゲーム難度 |

### 5-1. プリセット

| プリセット名 | speed | power | catch | kick | 特徴 |
|---|---:|---:|---:|---:|---|
| `speedy` | 9 | 4 | 6 | 3 | 足が速い |
| `power` | 4 | 9 | 5 | 3 | パワー型 |
| `hands` | 6 | 4 | 9 | 3 | 捕球がうまい |
| `kicker` | 5 | 5 | 4 | 9 | キックが得意 |
| `allround` | 6 | 6 | 6 | 6 | バランス型 |

直接指定も可能。

```js
{ speed: 7, power: 2, catch: 8, kick: 1 }
```

存在しないプリセット名を書いた場合は、`allround` 扱いにする。

### 5-2. 実装状態の注意

旧キャラクター差し替え指示書の時点では、`stats` のうち主に `speed` がランの走る速さの見た目に効いていた。

その後のゲーム性追記では、`speed`、`power`、`catch`、`kick` をプレー結果へ反映する方針になっている。したがって、仕様上は4能力すべてを保持し、実装では `gamelogic.js` 側で結果へ接続する。

## 6. 動物 `animal`

| 値 | 動物 | 見た目の特徴 |
|---|---|---|
| `owl` | フクロウ | 大きな目の縁、くちばし、羽角 |
| `rabbit` | ウサギ | 長い直立耳、内側ピンク、前歯 |
| `squirrel` | リス | 房毛耳、ほっぺ、大きな縞しっぽ |
| `bear` | クマ | 丸耳、茶色マズル、黒鼻 |
| `rhino` | サイ | 角、小さい耳、大きめマズル |
| `wolf` | オオカミ | とがった耳、先黒、白マズル、眉、白先しっぽ |
| `cat` | ネコ | 三角耳、内側ピンク、ピンク鼻、ひげ、縞しっぽ |

上記以外の文字列を指定した場合は、汎用の丸耳と鼻の見た目になる。

## 7. チーム編成

現在の初期メンバーは、ミミィ、ピョン太、ナッツ、ゴロ、ガードを味方攻撃側の仮メンバーとして扱う。マネージャーのミケ、相手側キャラのケリー等も仮メンバーであり、後から差し替え可能な前提にする。


### 7-1. 攻撃チーム

攻撃チームは5体固定。

例:

```js
var OFFENSE = [
  { name: "ミミィ",   role: "QB", animal: "owl",      fur: FUR.owl,      stats: "allround" },
  { name: "ピョン太", role: "RB", animal: "rabbit",   fur: FUR.rabbit,   stats: "speedy" },
  { name: "ナッツ",   role: "WR", animal: "squirrel", fur: FUR.squirrel, stats: "hands" },
  { name: "ゴロ",     role: "OL", animal: "bear",     fur: FUR.bear,     stats: "power" },
  { name: "ガード",   role: "OL", animal: "rhino",    fur: FUR.rhino,    stats: "power" }
];
```

### 7-2. 守備チーム

守備チームも5体固定。

- `role` は基本 `DF`
- 能力値は守備判定へ反映可能
- チームカラーで敵味方を見分ける

### 7-3. マネージャー

マネージャーは、選手とは別定義にする。

```js
var MANAGER = {
  name: "ミケ",
  fur: 0xf0e2c8,
  jacket: 0x2f9e8f
};
```

マネージャーは、役割・能力値を持たない。撮影・記録・分析の象徴として扱う。

## 8. チームカラー

```js
var TEAM = { home: 0xee8e3c, away: 0x3f7fd0 };
```

| 項目 | 意味 |
|---|---|
| `home` | 味方ユニフォーム色 |
| `away` | 相手ユニフォーム色 |

## 9. 差し替え手順

すべて `characters.js` を編集し、保存してブラウザを再読み込みする。

### 9-1. 名前だけ変える

```js
{ name: "新しい名前", role: "QB", animal: "owl", fur: FUR.owl, stats: "allround" }
```

### 9-2. 毛色を変える

```js
{ name: "ゴロ", animal: "bear", fur: 0x8a5a30 }
```

### 9-3. 動物を変える

```js
{ name: "ナッツ", role: "WR", animal: "wolf", fur: 0x9aa7b5, stats: "hands" }
```

### 9-4. 役割を変える

```js
{ name: "ピョン太", role: "P", animal: "rabbit", fur: FUR.rabbit, stats: "kicker" }
```

### 9-5. パラメータを変える

```js
{ name: "ゴロ", role: "OL", animal: "bear", fur: FUR.bear, stats: "speedy" }
```

### 9-6. 3要素を自由に組み合わせる

```js
{ name: "ダッシュ", role: "P", animal: "bear", fur: FUR.bear, stats: "speedy" }
```

### 9-7. ユニフォーム色を変える

```js
var TEAM = { home: 0xee8e3c, away: 0x3f7fd0 };
```

## 10. 重要制約

- `OFFENSE` と `DEFENSE` はそれぞれ必ず5体にする
- 攻撃チームには `QB`、`RB`、`WR` を1体ずつ入れる
- 色は必ず `0xRRGGBB` の数値にする
- `"#ee8e3c"` のような文字列は使わない
- `stats` のプリセット名は実在するものを使う
- 既存の動物見た目分岐は、必要がなければ変更しない
- 役割は並び順ではなく `role` で決まる

## 11. 新しい動物の追加

新種の見た目を足す場合は、`characters.js` の `makeAnimal()` に分岐を追加する。

### 11-1. パーツの置き方

```js
box(幅, 高さ, 奥行, 色, x, y, z)
```

座標:

- `y` が上
- `0` が地面
- `z+` が前
- `x` は左がマイナス、右がプラス

顔まわりは頭サブグループ `head` に追加する。

しっぽなど体に付くものは本体 `g` に追加する。

完成したキャラは自動で1.2倍に拡大される。

### 11-2. 追加例: ペンギン

```js
} else if (role === "penguin") {
  g.add(box(0.7, 1.0, 0.1, 0xffffff, 0, 1.35, 0.52));
  head.add(box(0.18, 0.12, 0.18, 0xf4a93c, 0, -0.14, 0.54));
  g.add(box(0.16, 0.7, 0.4, fur, -0.78, 1.35, 0));
  g.add(box(0.16, 0.7, 0.4, fur, 0.78, 1.35, 0));
}
```

追加後は、設定リストで `animal: "penguin"` と書く。

## 12. アート方針

- 丸っこくて可愛い
- ぽっちゃり動物
- ボクセル表現
- チームカラーで敵味方を見分ける
- 痛々しい表現を避ける
- 転んでもポヨン、コロコロ
- 教材用に役割ラベルを付けやすい見た目にする

## 13. 動作確認

1. `characters.js` を保存する
2. ブラウザで `index.html` を開く、または再読み込みする
3. 味方5体が意図した動物・色か確認する
4. 相手5体が表示されているか確認する
5. 名前がコーチのセリフに出るか確認する
6. テーピング発生時などに名前が反映されるか確認する
7. 画面が真っ白なら、カンマ抜け、`}` 閉じ忘れ、色の文字列指定を疑う
8. ブラウザの開発者ツールのコンソールを確認する
