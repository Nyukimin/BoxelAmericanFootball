# 05_采配・読み合い仕様書

- 目的: 攻撃・守備の采配、CPU傾向、読み合い、結果解説を定義する
- 関連: `02_ゲームコンセプト仕様書.md`, `04_試合進行・ルール仕様書.md`

## 1. 基本方針

本作の中心は「状況を見て、作戦を選び、結果から学ぶ」ことである。

采配は、ゲーム性と教材性の両方を支える。

- ゲーム性: 相手の読みを外し、作戦が決まる快感
- 教材性: なぜその作戦が効いたかを理解する学び

## 2. 攻撃采配の流れ

```text
現在状況を見る
  ↓
隊形を選ぶ
  ↓
プレーを選ぶ
  ↓
スナップ
  ↓
結果を見る
  ↓
記録・解説を見る
```

## 3. 攻撃隊形

| 隊形 | 見た目のイメージ | 狙い・特徴 | 向いているプレー |
|---|---|---|---|
| ショットガン | QBが少し後ろに構える | 視野が広く、パスを狙いやすい花形の隊形 | ショートパス、ロングパス |
| アイフォーメーション | 走る選手が縦に一列 | 正面へ力強く突進する力押しの隊形 | ラン |
| シングルバック | バランスのよい配置 | ランもパスも狙える万能型。読まれにくい | ラン、パス両方 |
| ゴールライン | 全員で密集 | あと1〜2ヤードを押し込む接近戦用 | ショートラン |

隊形を選ぶと、クオータービュー上で実際に選手の並びが変わる。初心者が見た目で違いを理解できるようにする。

## 4. 攻撃プレー

| プレー | 内容 | リスクと見返り |
|---|---|---|
| ラン | ボールを持って走り、地上で前進する | 確実だが大きくは進みにくい |
| ショートパス | 近くの選手へ短く投げる | 成功しやすく、着実に前進 |
| ロングパス | 遠くの選手へ大きく投げる | 決まれば一気に前進。失敗・奪われる危険もある |

4thダウンでは、状況に応じて以下も選べる。

- フィールドゴール
- パント
- 勝負継続

## 5. 状況判断の基本

| 状況 | 判断例 | 教材ポイント |
|---|---|---|
| 1st & 10 | 安全に始める、相手を見る | 最初の一手 |
| 2nd & short | 攻めた選択もできる | 残り距離の余裕 |
| 3rd & short | 確実に進めたい | ファーストダウン意識 |
| 3rd & long | パスを狙うか、次に備える | リスク判断 |
| 4th & short | 勝負かキックか | 攻守交代の重み |
| 4th & long | パントやFGを考える | 陣地と得点の判断 |
| ゴール前 | ランの押し込みが強い | ゴールラインの考え方 |

## 6. 守備采配

味方が守備のとき、攻撃と同じように2段階カードで選ぶ。

### 6-1. 守備隊形

| 守備隊形 | 狙い |
|---|---|
| ラン重視 | 中央を厚くしてランを止める |
| バランス | ランにもパスにも対応する |
| パス重視 | 後ろを厚くしてパスを止める |
| ゴールライン | ゴール前・短距離を止める |

### 6-2. 守備戦術

| 戦術 | 狙い | リスク |
|---|---|---|
| 通常 | 安定して守る | 大きな強みはない |
| ブリッツ | 強く前へ出てターンオーバーを狙う | 外すと大きく進まれやすい |
| ゾーン | 大失点を抑える | 一発の奪取力は低め |

## 7. CPU傾向

CPUは隠れ傾向を持つ。

```js
pickCpuStyle() -> "run" | "pass" | "balanced"
```

| 傾向 | 意味 |
|---|---|
| `run` | ラン寄りの攻撃をしてくる |
| `pass` | パス寄りの攻撃をしてくる |
| `balanced` | ラン・パスを混ぜる |

将来は、スナップ前の守備や攻撃の「気配」として見せる。

## 8. 守備結果判定

守備結果は `defenseDrive` で処理する。

```js
defenseDrive({
  front,
  tactic,
  los,
  pinBonus,
  dPower,
  dSpeed,
  cpuStyle
}) -> {
  points: 0 | 3 | 7,
  turnoverWon: boolean,
  msg: string
}
```

判定要素:

- フィールド位置
- 守備隊形とCPU傾向の相性
- 守備能力
- 戦術補正
- パントによる押し込み補正

結果:

- 止める: 0点
- FGを許す: 相手+3点
- TDを許す: 相手+7点
- ターンオーバー奪取: 味方ボール

## 9. 読み合いの対応関係

| 守備隊形 | 刺さる相手傾向 | 弱い相手傾向 |
|---|---|---|
| ラン重視 | `run` | `pass` |
| パス重視 | `pass` | `run` |
| ゴールライン | 短距離ラン | 深いパス |
| バランス | `balanced` | 決定的な強みはない |

| 戦術 | 効果 |
|---|---|
| ブリッツ | ターンオーバー奪取率上昇。外すと手薄 |
| ゾーン | TDを抑えやすい。堅実 |
| 通常 | 補正なし。安定 |

## 10. 能力値との接続

| 能力 | 攻撃での意味 | 守備での意味 |
|---|---|---|
| `speed` | ランの伸び、YAC | カバー、インターセプト |
| `power` | 押し込み、ファンブル耐性 | ラン阻止、奪取 |
| `catch` | 捕球 | 将来の守備捕球にも利用可能 |
| `kick` | FG、パント | 守備では基本使わない |

## 11. 守備のテル

守備のテルとは、スナップ前に守備が何を警戒していそうかを画面で示す情報である。

例:

- 中央に密集: ラン警戒
- 後ろに広く配置: パス警戒
- 前に飛び出し気味: ブリッツの気配

テルは、初心者に読み合いを体験させるための教材UIでもある。完全に正解を見せるのではなく、少しだけ手がかりを出す。

## 12. コーチの「なぜ」解説

毎プレー後、結果の理由を一言で伝える。

例:

| 結果 | 解説例 |
|---|---|
| ラン成功 | 「ラインが押して、走る道ができたね。」 |
| ラン失敗 | 「相手が中央に集まっていたから、走るのは止められたね。」 |
| ショートパス成功 | 「近くの受け手が空いていたから、確実に通ったよ。」 |
| ロングパス失敗 | 「遠くを狙ったぶん、失敗しやすいプレーだったね。」 |
| インターセプト | 「相手がパスを読んでいたよ。」 |
| 守備成功 | 「ランを読んで中央を厚くしたのが効いたね。」 |
| 守備失敗 | 「パス警戒が薄くて、大きく進まれたよ。」 |

## 13. マネージャー向け記録との接続

采配結果は、記録・分析表示へつなげる。

例:

```text
プレー: シングルバック / ラン
結果: +4yd
状況: 2nd & 6
理由: 中央が少し空いた
見るポイント: ラインの押し込み
```

記録表示は、ゲーム内の判断材料であり、マネージャー教材でもある。

## 15. 制約

- 攻撃も守備も、初心者がカード説明だけで選べるようにする
- 守備采配は現状、1相手ポゼッション=采配1回とする
- 相手オフェンスを全員動かす可視化は後段
- 複雑な戦術名を最初から増やしすぎない
- 読み合いを入れるが、理不尽な負け方に見せない
