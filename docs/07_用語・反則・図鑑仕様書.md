# 07_用語・反則・図鑑仕様書

- 目的: アメフト用語、反則、初出解説、用語図鑑を定義する
- 関連: `03_学習カリキュラム仕様書.md`

## 1. 基本方針

本作では、アメフトのルール用語をゲーム中の出来事として登場させ、初出で短く解説する。

目的は、用語を暗記させることではない。マネージャーが現場で聞く言葉に慣れ、試合中に何が起きたか分かるようにすることである。

## 2. 用語表示の原則

1. 用語はイベントに紐づけて出す
2. 初出のみ丁寧に説明する
3. 2回目以降は短くする
4. 用語図鑑に自動登録する
5. 詳細確認は任意にする
6. テンポを止めない

## 3. 反則システム

### 3-1. 目的

反則をゲーム内で低頻度に発生させ、フラッグ、罰退、自動ファーストダウンなどの基本概念を学べるようにする。

### 3-2. 表現

- 審判が黄色いフラッグを投げる
- ホイッスル先生が一言説明する
- 罰退方向を矢印で見せる
- 該当ダウンをやり直す、または前進する
- 有利な側が得する方を自動採用する

### 3-3. 頻度

- 低頻度
- 1試合に数回程度
- 初心者のテンポを壊さない
- 反則が主役にならない

## 4. 反則一覧

| 反則 | 誰が | きっかけ | 効果 |
|---|---|---|---|
| フォルススタート | 攻撃 | スナップ前に味方が動く。低確率 | 5ヤード罰退・ダウンやり直し |
| オフサイド | 守備 | スナップ前に相手が出る。低確率 | 攻撃に5ヤード・ダウンやり直し |
| ホールディング | 攻撃/守備 | ラインやカバーで掴む。プレー中低確率 | 攻撃10ヤード罰退 / 守備5ヤード + 自動ファーストダウン |
| パスインターフェア | 守備 | ロングパス時に受け手を妨害。低確率 | 反則地点まで前進 + 自動ファーストダウン |
| フェイスマスク | 守備 | タックル時にマスクを掴む。低確率 | 15ヤード |
| アンネセサリーラフネス | 守備 | 乱暴なタックル。低確率 | 15ヤード + 自動ファーストダウン |
| ディレイオブゲーム | 攻撃 | プレイクロック切れ。時間系導入後 | 5ヤード |

## 5. 反則処理ルール

### 5-1. 受諾/拒否

初心者に複雑さを出さないため、反則の受諾/拒否は自動処理する。

有利な側が得する方を自動で採用する。

### 5-2. 可視化

- 攻撃の反則: 自陣方向へ下がる矢印
- 守備の反則: 前進方向へ進む矢印
- 自動ファーストダウン: 目標ライン更新を強調

### 5-3. 解説

毎回、用語を短く説明する。

例:

```text
黄色いフラッグ！ 今のはオフサイド。守備が早く出すぎた反則だよ。攻撃は5ヤード進むよ。
```

## 8. 用語図鑑

### 8-1. 目的

出会った用語が図鑑に自動で埋まり、あとから復習できる。

教材効果とコレクション欲を両立する。

### 8-2. 表示仕様

- 発見済み用語: 通常表示
- 未発見用語: グレー表示
- 用語ごとに短い説明
- 発生したプレー例へのリンクを将来追加可能
- カテゴリ別に整理する

### 8-3. カテゴリ

- 得点
- ダウン・攻守
- 攻撃プレー
- 守備
- スペシャルチーム
- 反則
- 時間
- フィールド・ポジション

## 9. 実装段階

優先順:

1. 反則システム
2. フラッグ、罰退、自動ファーストダウンの可視化
3. 初出解説
4. 追加点、2ポイント、セーフティ、タッチバック、フェアキャッチ、キックオフ、サック、ファンブルの発生
5. クォーター、2ミニッツ警告、タイムアウト、プレイクロック
6. 用語図鑑

## 10. TDD方針

反則判定、罰退、やり直し、自動ファーストダウンは純粋関数化する。

例:

```js
resolvePenalty({ foul, side, down, toGo, los }) -> {
  los,
  down,
  toGo,
  automaticFirstDown,
  replayDown,
  message
}
```

確率系は低頻度であることを大数テストで確認する。

## 11. 制約

- 反則は多発させない
- 公式ルールの細部は省略する
- 受諾/拒否は自動
- 初心者が混乱しないよう、用語説明は短くする
- 反則がゲームのテンポを壊さないようにする
- 痛々しい表現はしない
